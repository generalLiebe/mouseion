# Mouseion トークン経済設計

## 概要

Mouseionトークン（仮称: $MUSE）は、エコシステムの基盤として、報酬分配・ガバナンス参加・サービスアクセスを可能にします。

## トークンの役割

### 1. 報酬支払い
- 貢献スコアに基づいて貢献者に分配
- AI学習・生成におけるデータ利用への対価
- 専門家アノテーションへの報酬

### 2. ガバナンス権
- プロトコル変更への投票
- パラメータ調整への参加
- システムアップグレードの承認

### 3. ステーキング
- 専門家/ガーディアンの担保要件
- 品質保証のための評価ステーキング
- 不正行為に対するスラッシング

### 4. サービスアクセス
- AIサービス利用料
- マーケットプレイスでの購入
- プレミアム機能へのアクセス

## 収益源

報酬プールの持続可能性は、多様な収益源に依存します：

| 収益源 | 説明 |
|:--|:--|
| AI利用料 | AIサービス利用時の課金 |
| データ販売手数料 | マーケットプレイスでの取引手数料 |
| プレミアム機能 | 高速処理、優先アクセス等 |
| 企業向けライセンス | 大規模利用者向けの契約 |

## トークン配布設計（案）

| 配布先 | 割合 | ロック期間 |
|:--|:--|:--|
| 初期貢献者（データ提供者） | 40% | 段階的解除 |
| 開発チーム | 15% | 2〜4年 |
| エコシステム基金 | 20% | - |
| 流動性提供 | 15% | - |
| コミュニティ報酬 | 10% | - |

**注**: これらは暫定的な数値であり、コミュニティでの議論に基づいて変更されます。

## インフレーション設計

### 新規発行
- 報酬支払い用に年間X%の新規発行
- ガバナンスによるレート調整可能

### バーン機構
- 手数料の一部を焼却してインフレ抑制
- 利用増加に伴うデフレ圧力

### 調整メカニズム
- 発行レート変更のガバナンス投票
- ガーディアン承認による緊急調整

## 報酬分配の計算式

### 貢献スコア計算

```
貢献スコア = (利用頻度 × w1) +
            (品質スコア × w2) +
            (類似度スコア × w3) +
            (希少性係数 × w4)
```

各パラメータ：
- `w1, w2, w3, w4` はガバナンスで決定される重み
- デフォルト: w1=0.3, w2=0.3, w3=0.25, w4=0.15

### 報酬計算

```
個人報酬 = (貢献者スコア / 全スコア合計) × 報酬プール
```

## ステーキング要件

### 専門家
- 最低ステーク量: 未定
- スラッシング条件:
  - 継続的な低品質アノテーション
  - 共謀の検出
  - 非活動ペナルティ

### ガーディアン
- より高い最低ステーク量が必要
- スラッシング条件:
  - 誤った不正判定
  - 共謀
  - 応答遅延

## 大口保有者対策

### ガバナンス
- アドレスあたりの最大投票力
- 重要な決定には二次投票
- 貢献度加重投票オプション

### 経済面
- 段階的な手数料構造
- 小口貢献者向け無料枠
- 大口保有者の監視

## トークン価値の考慮事項

### 価値ドライバー

- 品質データセットへの独占アクセス
