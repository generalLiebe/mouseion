# Mouseion ホワイトペーパー

**バージョン 0.1 - ドラフト**

## 概要

Mouseionは、AIシステムへの貢献を追跡し、全ての参加者に公正に報酬を分配する分散型プロトコルです。新しい可逆送金型ブロックチェーンを活用し、透明で監査可能かつ回復可能な支払いフローを実現しながら、貢献者のデータ主権を維持します。

---

## 1. はじめに

### 1.1 ビジョン

AIの発展によって生まれる価値を、データ提供者・クリエイター・レビュワー・AI開発者など全ての貢献者に公正に還元する分散型エコシステムを構築する。

### 1.2 解決する課題

現行のAIシステムには根本的な問題があります：

1. **報酬の不在**: AIモデルの学習に貢献したデータ提供者に適正な報酬が還元されていない
2. **透明性の欠如**: どのデータがAIの学習・生成に使用されたか追跡できない
3. **権利の侵害**: 著作権者の意思に関係なくデータが利用される
4. **中央集権的支配**: 少数の企業がAIシステムを独占的に管理している
5. **データの喪失**: サービス終了やアカウント削除によりデータが失われる

### 1.3 提供価値

Mouseionが実現すること：

- **データ主権**: データ提供者が自分のデータに対する主権を持てる
- **公正な報酬**: 貢献度に応じた透明な報酬分配
- **民主的ガバナンス**: 貢献者全員がシステム運営に参加できる
- **永続的なデータ保存**: ブロックチェーン上でデータが永続する
- **トレーサビリティ**: 生成物の出典が追跡可能

## 2. システムアーキテクチャ

### 2.1 レイヤー構成

```
┌─────────────────────────────────────────────────────────────┐
│  アプリケーション層                                          │
│  ・ウォレットUI  ・データ登録画面  ・報酬確認画面            │
│  ・ガバナンス投票画面  ・マーケットプレイス                  │
├─────────────────────────────────────────────────────────────┤
│  サービス層：AI印税分配システム                              │
│  ・貢献追跡  ・報酬分配  ・ガバナンス  ・データ管理          │
├─────────────────────────────────────────────────────────────┤
│  レイヤー1：可逆送金型ブロックチェーン（Julia）              │
│  ・保留型送金  ・取消機能  ・ガーディアン承認                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 基盤チェーンとの連携

本システムは「可逆送金型ブロックチェーン」の上に構築されます：

- **誤送金の防止**: 報酬分配時の宛先ミスを取り消し可能
- **詐欺対策**: 不正な報酬請求をガーディアンが審査・凍結可能
- **保留型送金**: 報酬受取の確認プロセスを組み込み可能

## 3. コア機能

### 3.1 データ登録とタグ付け

#### 登録時のメタデータ

| 項目 | 説明 |
|:--|:--|
| データ本体のハッシュ | 改ざん検知用 |
| AI学習許可フラグ | オプトイン/オプトアウト |
| 販売可否フラグ | データ自体の販売を許可するか |
| 利用条件 | 商用利用可否、クレジット表記要否など |
| カテゴリ・タグ | 専門分野、データ種別 |
| 提供者の公開鍵 | 報酬受取先 |

#### ストレージ設計

- **オンチェーン**: メタデータ、ハッシュ、権利情報、貢献記録
- **オフチェーン**: データ本体（IPFS、Arweave等の分散ストレージ）

### 3.2 貢献者特定アルゴリズム

AIが出力を生成した際、どのデータが寄与したかを特定する手法：

| 手法 | 用途 | 特徴 |
|:--|:--|:--|
| Sentence-BERT / コサイン類似度 | 意味的類似性の計測 | テキスト間の類似度評価 |
| TF-IDF / Word2Vec | キーワード・タグ検索 | プロンプトとデータの関連性 |
| クラスタリング（K-Means, DBSCAN） | パターン識別 | 特徴ベクトルによる分類 |
| RAG | 検索・生成の組み合わせ | 参照データの明示的な特定 |
| Attention Weight解析 | Transformerモデル | どの入力に注目したか |
| Influence Functions | 勾配ベースの寄与度計算 | 学習データの影響度推定 |

#### 技術的課題と対策

- **計算コスト**: 事前インデックス化、近似最近傍探索（ANN）の活用
- **精度の限界**: 複数手法の組み合わせ、閾値の設定
- **リアルタイム性**: ユースケースに応じたバッチ/リアルタイム選択

### 3.3 報酬分配システム

#### 分配フロー

```
1. AIサービス利用者が利用料を支払う
2. システムが利用されたデータの貢献者を特定
3. 貢献度に応じてトークンを分配
4. 貢献者のウォレットに報酬が保留状態で送金
5. 貢献者が受領確認（基盤チェーンの保留型送金機能）
```

#### 貢献度の計算要素

- **データの利用頻度**: 何回参照されたか
- **データの品質スコア**: 専門家評価、コミュニティ評価
- **類似度スコア**: 生成物との関連性の強さ
- **希少性**: 代替データの有無

### 3.4 専門家によるアノテーション・評価

#### 専門家選定プロセス

1. 審査区分表（学術分野・産業領域で分類）に基づきリスト化
2. 過去の貢献度・評価履歴・専門性スコアを算出
3. AIアルゴリズム＋コミュニティ投票で選定
4. 選定された専門家がアノテーションを実施

#### 品質保証

- 他のレビュワーによる相互評価
- 評価の一致度によるアノテーション信頼性スコア
- 不正評価の検知と制裁

### 3.5 AIモデルのバージョン管理

#### 記録項目

- モデルID、バージョン番号
- 学習に使用したデータセットのハッシュ
- 学習パラメータ
- 変更履歴（diff）
- 公開日時

#### 目的

- サイレントな性能変動の防止
- ユーザーが特定バージョンを選択可能
- 問題発生時のロールバック

### 3.6 人間データとAI生成データの分離

#### 必要性

- AI生成データで学習したAIは品質低下のリスク（モデル崩壊）
- 人間のオリジナルデータの価値を保護

#### 実装方法

- 登録時に「人間作成」「AI生成」「AI支援」のタグを必須化
- AI生成データは別インデックスで管理
- 学習時にフィルタリング可能

## 5. ガバナンス設計

### 5.1 議決権の行使範囲

- AI利用ルールの策定・変更
- データ公開・非公開ポリシー
- 手数料率の設定
- システムアップグレードの承認
- 審査区分表の更新
- 報酬分配アルゴリズムの変更
- システム保有資産の運用

### 5.2 投票メカニズム

| 方式 | 特徴 | 用途 |
|:--|:--|:--|
| 1トークン1票 | シンプル | 一般的な提案 |
| 二次投票（Quadratic Voting） | 大口支配の緩和 | 重要な意思決定 |
| 委任投票 | 専門家への委任可能 | 技術的判断 |
| 時間加重投票 | 長期保有者を優遇 | 長期的な提案 |

### 5.3 大口保有者支配の防止策

- **投票力の上限**: 1アドレスあたりの最大投票力を制限
- **二次投票**: 票数の平方根で影響力を計算
- **貢献度加重**: トークン量だけでなく実際の貢献度も考慮
- **クールダウン期間**: 大量購入直後の投票を制限

## 6. 法務・コンプライアンス

### 6.1 著作権法との関係

- オプトイン方式により、明示的な許諾を得たデータのみを利用
- 著作権者がデータ利用条件を細かく設定可能
- 利用履歴をブロックチェーンに記録し、紛争時の証拠として活用

### 6.2 データ保護規制

- GDPR対応: オフチェーンデータは削除可能、オンチェーンはハッシュのみ
- プライバシーオプション: 公開鍵のみで活動可能
- データ削除リクエストへの対応プロセス

### 6.3 金融規制

- ユーティリティトークンとしての設計で証券該当性を回避
- 高額取引にはアプリ層でKYC/AML対応
